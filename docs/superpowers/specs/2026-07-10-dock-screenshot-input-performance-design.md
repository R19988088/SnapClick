# SnapClick Dock、截图输入与性能优化设计

日期：2026-07-10

## 目标

在不减少现有功能、不改变 Dock 多窗口缩略图尺寸与加载行为的前提下，完成以下改进：

1. 遮住 Dock 自带的应用标题气泡，为缩略图面板增加指向当前 Dock 图标的尖角。
2. Dock 缩略图面板始终使用图标最大放大状态对应的高度，避免遮挡未放大的图标。
3. 修复窗口截图缺少窗口描边和投影、设置开关无效的问题。
4. 截获 `Alt+A`，不再向前台应用输入 `å`。
5. 为截图标注画笔加入低延迟的压力稳定和坐标防抖。
6. 将截图工具栏改为原生玻璃材质的圆角矩形，并替换不合适的图标。
7. 优化截图、标注和图像输出的内存峰值与热路径性能。

## 不变合同

- Dock 仍以每个真实窗口为一个可点击缩略图。
- Dock 缩略图的宽高、窗口枚举、异步抓图和关闭/激活行为保持不变。
- 被遮挡窗口点击 Dock 图标时只前置；真正位于前台时才最小化。
- 截图选区外仍保持暗化，保存和复制仍统一经过 `ScreenCaptureEngine.applyScreenshotEffects(to:)`。
- 高亮工具仍通过降低未选区域的对比度和饱和度实现，不使用灰色遮罩。
- 每个标注工具继续独立保存尺寸，白色和黑色仍保留在颜色预设中。
- 本轮不新增第三方依赖。

## 方案选择

采用现有 AppKit 架构内的定点修复，不重写 Dock 控制器或截图编辑器。

未采用整体重写，因为当前 Dock 多窗口布局已经稳定，重写会重新扩大窗口枚举、点击控制和几何布局的回归面。未采用只调偏移和阴影参数的表面修复，因为热键事件传播、窗口捕获降级和标注重绘仍会保留根因。

## Dock 面板

### 固定最大高度

当前面板直接使用命中图标的 `bounds.maxY`。Dock 放大动画尚未发生时，该值较低，导致面板覆盖图标。

底部 Dock 的面板基线改为以下两者的最大值：

- 当前命中图标的顶部；
- Dock 配置中的最大图标尺寸 `largesize` 对应的稳定顶部。

左侧和右侧 Dock 使用相同规则计算稳定水平基线。Dock 自动隐藏时仍以当前命中图标位置兜底。面板横向或纵向中心继续跟随当前图标，并保留屏幕边界夹取。

### 尖角和系统标题

macOS 没有公开 API 可关闭 Dock 进程生成的标题气泡，因此不修改 Dock 偏好或辅助功能属性。

缩略图面板改为带方向尖角的单一背景：

- 底部 Dock：尖角朝下。
- 左侧 Dock：尖角朝左。
- 右侧 Dock：尖角朝右。
- 尖角中心对准命中图标中心，并避开面板圆角。

面板使用弹出菜单层级覆盖 Dock 标题气泡。尖角区域与玻璃主体使用同一视觉材质，从视觉上完整遮住系统标题。缩略图内容区和 tile 尺寸不变。

## 窗口描边和投影

### 根因

`captureSingleWindow(_:)` 的旧 CGWindow 路径可能返回带系统 framing 的图像，但 ScreenCaptureKit 降级只返回窗口内容。后处理又只能接收普通 `NSImage`，不知道图像是否已经包含系统 framing，因此“添加阴影”开关不能稳定决定最终结果。

### 统一输出合同

窗口截图先尝试获得带系统外框和投影的图像。若平台只返回窗口内容，则由共享后处理补齐：

- 1pt、随外观调整透明度的窗口分隔描边；
- macOS 风格的向下柔和投影；
- 保留原始透明度和圆角；
- 投影开关关闭时不生成额外投影；
- 圆角开关继续只控制输出圆角，不再承担“窗口透明”这一含混语义。

区域截图若是普通手动矩形，描边和投影作用于最终矩形输出。若由窗口命中进入标注，则使用窗口 framing 路径，保证编辑预览、复制、保存和导出一致。

设置页将“窗口透明”改为与实际行为一致的“添加圆角”，并保留当前开关值和存储键，避免迁移破坏用户设置。

### 性能约束

移除 `visiblePixelBounds(in:)` 的整图 RGBA 分配和逐像素扫描。后处理直接使用捕获阶段已知的内容矩形；普通区域截图使用完整图像矩形。这样可避免 4K 截图在输出时额外分配约 32 MB 缓冲并执行一次全图 CPU 扫描。

## Alt+A 热键

事件 tap 在同步匹配阶段判断快捷键。匹配成功后立即返回 `nil`，同时把 action 派发到主线程。

`handleKeyEvent` 改为返回是否命中，保证只有已注册且修饰键完全匹配的快捷键被吞掉。未匹配按键、自动重复事件和事件 tap 自愈事件继续透传。截图启动速度不再影响字符是否输入。

## 压力输入和笔触稳定

### 输入采样

画笔拖动时按时间顺序处理 AppKit 提供的合并鼠标/数位板事件；若系统没有合并事件，则处理当前事件。每个样本立即产出，不等待固定窗口填满。

普通鼠标没有有效压力时使用当前工具尺寸，避免将零压力误算为零线宽。数位板压力保持 `0...1`，再映射到现有最大 `1.8x` 线宽。

### 精简稳定器

采用两条独立、固定状态的因果滤波链：

- 压力：最多 4 个有效样本的移动平均。首包立即输出，起笔和抬笔清空历史。
- 坐标：速度自适应指数平滑。慢速时加强抖动抑制，快速移动时提高权重并接近原始坐标。

稳定器不缓存等待未来样本、不启动计时器、不分配每包对象。现有按距离补点逻辑继续负责线段密度，但改为使用稳定后的坐标和压力。

本轮不新增防抖设置项。参数使用内部保守默认值，避免扩大 UI 和持久化范围；后续只有在实际测试需要用户调节时再暴露设置。

### 绘制与重绘

画笔仍使用逐段可变线宽和圆头连接。拖动时只标记上一点到新点的扩展脏矩形，不再对每个输入样本设置整张画布 `needsDisplay = true`。高亮、形状工具和选区变化仍按现有需要完整重绘。

## 工具栏视觉

工具栏主体保持现有两行信息结构和所有功能，但从胶囊改为圆角矩形：

- 主体连续圆角 12pt。
- 工具按钮连续圆角 7pt，不再使用圆形按钮背景。
- macOS 26 使用 `NSGlassEffectView`/`NSGlassEffectContainerView`。
- 旧系统使用 `NSVisualEffectView`，材质和边框作为兼容回退。
- 阴影收敛为较小半径，避免当前大面积悬浮胶囊感。

图标继续统一使用 SF Symbols，不绘制自定义 SVG：

- 画笔：`pencil.tip`
- 文字：`textformat`
- 高亮：`rectangle.dashed`
- 马赛克：`square.grid.3x3.fill`
- 序号：`number.circle`
- 拖动：`arrow.up.and.down.and.arrow.left.and.right`

若当前系统不支持某个符号，回退到同工具现有符号。按钮 tooltip、快捷键和工具行为不变。

## 其他性能与内存优化

Dock 缩略图性能不在本轮优化范围。

### 高亮缓存

`AnnotationCanvas` 在 `baseImage` 变化时生成一次低对比度图并缓存。高亮拖动和画布重绘只复用缓存，不再每帧执行 `CIColorControls` 和 `createCGImage`。底图替换时立即释放旧缓存。

### Core Image 上下文

截图和标注共用长生命周期 `CIContext`。删除每次样本转换时临时创建 context 的路径，降低 Metal/颜色管理资源反复初始化。

### 图像编码

保存截图直接从 `CGImage` 创建 `NSBitmapImageRep`，删除 `NSImage -> TIFF Data -> NSBitmapImageRep -> PNG/JPEG` 中间往返，降低高分辨率截图的峰值内存和复制次数。

### 标注路径

起笔时为坐标和压力数组预留小型初始容量，减少连续绘制中的数组扩容。完成、撤销和清空后仍由现有数据模型释放对象，不新增长期缓存。

### 任务生命周期

截图覆盖层关闭时取消仍在运行的窗口捕获和长截图任务，并断开大图引用。所有缓存均绑定到单个编辑会话，不建立无上限的全局图片缓存。

## 错误处理

- 系统原始窗口 framing 捕获失败时自动使用共享描边/投影，不中断截图。
- 原生玻璃类不可用时回退 `NSVisualEffectView`。
- 合并输入事件不可用时使用当前事件。
- Core Image 低对比处理失败时显示原始底图，不应用灰色覆盖。
- 热键 action 启动失败不把已吞掉的 `Alt+A` 重新注入系统。

## 测试与验收

先添加会失败的契约测试，再修改实现。

自动检查覆盖：

- 已注册热键命中返回 consumed，未命中事件透传。
- Dock 面板位置使用最大 Dock 图标尺寸，尖角方向与 Dock orientation 一致。
- Dock 缩略图尺寸、枚举和抓图函数未改变。
- 阴影关闭不生成投影，开启时生成描边和投影。
- 窗口 framing 降级仍产生一致输出。
- 压力平均首包立即输出，重置后不继承上一笔历史。
- 坐标滤波快速运动趋近原始输入，慢速抖动得到抑制。
- 工具栏圆角和玻璃回退合同一致。
- 高亮低对比图不会在每次 `draw(_:)` 中重新生成。
- 保存路径不再经过 TIFF 中间编码。

人工验收覆盖：

- Dock 图标未放大和已放大两种状态下，面板底边高度一致且不遮挡图标。
- 系统标题气泡不可见，尖角准确指向当前图标。
- 底部、左侧和右侧 Dock 均可用。
- `Alt+A` 在文本输入框中不会留下 `å`。
- 窗口截图在设置开关开启时包含清晰描边和投影，关闭时无额外投影。
- 数位板慢速画线抖动减少，快速画线无明显拖尾，压力粗细连续。
- 工具栏在浅色、深色及 macOS 26 玻璃环境下图标清晰、无重叠。
- 高亮连续拖动、4K 截图保存和多次截图后内存能回落。

最终验证顺序：契约脚本、Debug 编译、签名 Release 构建、应用签名验证、DMG 打包与验证、SHA-256 记录。未收到推送要求时不推送远端。
