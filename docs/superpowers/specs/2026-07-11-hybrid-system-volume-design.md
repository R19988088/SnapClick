# 系统音量控制混合架构设计

## 目标

让固定音量 HDMI/DisplayPort 输出在 macOS 控制中心显示可拖拽音量控制点；如果系统控制点因系统版本或设备行为不可用，SnapClick 状态栏下拉菜单仍提供同步的软件音量滑条。内建扬声器继续使用系统原生音量，默认麦克风绝不改变。

## 已排除方案

- 仅使用 Core Audio Process Tap：可以截获和衰减音频，但不能给物理显示器注入 `kAudioDevicePropertyVolumeScalar`，控制中心不会出现拖拽点。
- 仅使用普通虚拟声卡回读：之前已经出现无声、输出切换卡死和输入设备异常，不能恢复这条数据链。
- 修改显示器物理设备属性：Core Audio 不允许其他插件给现有设备追加可写控制属性。

## 架构

### SnapClick Control Output

安装一个仅输出的 AudioServerPlugIn：

- 双声道 Float32 输出，不发布输入流或输入设备。
- 发布主音量和静音控制，使其成为默认输出时控制中心显示可写音量滑条。
- 驱动只保存控制值和接收输出，不承担送往物理设备的主转发职责。
- Bundle、设备 UID 和签名固定，安装与卸载均需要管理员授权。

### 原生音频转发

SnapClick 启用软件音量时：

1. 保存当前真实默认输出 UID，不读取或修改默认输入。
2. 创建排除 SnapClick 自身的全局 Process Tap，并设置 `CATapMutedWhenTapped`。
3. 创建私有 Aggregate Tap 和捕获 IOProc。
4. 启动物理 AUHAL 输出到保存的真实设备。
5. 将默认输出切换到 SnapClick Control Output，使系统控制中心绑定虚拟设备的音量控制。
6. 监听虚拟设备音量和静音属性，把它们同步到转发环形缓冲区的原子增益。

停用、退出、输出设备消失或启动失败时，先恢复真实默认输出，再销毁 AUHAL、Aggregate 和 Tap。任何失败路径都不得修改默认输入。

## 输出切换

- 用户在系统声音设置中选择真实输出时，将该设备记为新的物理目标，然后重建一次 AUHAL，最后重新选择 SnapClick Control Output。
- 忽略由 SnapClick 自己切换到虚拟输出产生的默认输出通知，避免重建循环。
- 使用串行状态机合并重复通知；同一时刻只允许一个启动、停止或切换操作。
- 内建扬声器作为物理目标时，虚拟控制值仍驱动软件增益；停用功能后立即恢复内建扬声器的原生控制。

## 双向音量同步

- 控制中心修改虚拟设备主音量或静音时，SnapClick 更新 `gain`、`lastNonzeroGain` 和菜单滑条。
- 菜单滑条修改时，SnapClick 写入虚拟设备音量属性，并立即更新软件增益。
- 实体音量键由虚拟默认输出的标准控制处理；HID Tap 只作为驱动未就绪时的后备，避免一次按键调节两次。
- 音量范围为 `0...1`，静音恢复上一次非零值。

## 状态栏菜单

在现有 SnapClick 下拉菜单增加一个紧凑音量区：扬声器图标、单条横向 Slider 和百分比。功能未启用时禁用；启用但驱动控制监听失败时，Slider 仍直接调用软件增益，保证外接显示器可调。

## 兼容性与安全恢复

- Process Tap 仍要求 macOS 14.2；低版本继续显示灰色 12.5px 说明。
- 驱动只包含输出范围，合约测试禁止任何输入 scope、输入流或默认输入写入。
- 启动前和停止后验证默认输入 UID 未变化。
- App 异常退出后，下次启动先检测默认输出是否为 SnapClick Control Output；若是，先恢复保存的真实输出，再决定是否重新启用。
- 卸载顺序固定为：恢复真实输出、停止转发、删除驱动、重启音频服务、再次确认默认输入。

## 测试与验收

- 驱动合约：双声道输出、主音量、静音、无输入设备/流。
- 控制器合约：默认输出切换顺序、通知抑制、音量双向同步、所有失败路径不写默认输入。
- UI 合约：状态栏存在单条音量 Slider，设置页保留启用/停用和低版本说明。
- 真实设备：MacBook 扬声器、BenQ DisplayPort、控制中心拖拽、键盘音量键、菜单 Slider、输出热切换、退出恢复和麦克风/输入法语音。
- 发行：签名 Release、App/扩展/驱动同团队签名、双架构、DMG 校验和 SHA-256。
