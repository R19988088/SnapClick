# 输入法重启与音频恢复设计

## 目标

在“设置 → 其他”提供当前第三方输入法的重启操作，并继续修复 SnapClick Audio 的外接显示器音量控制。音频安装、启用、停止和卸载不得改变或破坏用户的默认麦克风；故障恢复必须能在 SnapClick 主程序无法正常退出时执行。

## 当前输入法识别

- 使用 Text Input Sources API 读取当前选中的键盘输入源。
- 读取输入源的 Bundle ID、显示名和 Bundle 路径。
- 仅把 `/Library/Input Methods` 或 `~/Library/Input Methods` 下的 App 视为可重启的第三方输入法。
- 系统自带输入法不显示重启按钮，避免终止 macOS 系统组件。
- 输入源变化时刷新显示，使设置页始终对应当前输入法。

## 输入法重启行为

- 在“其他”卡片中增加“输入法”行，右侧显示当前第三方输入法名称和“重启输入法”按钮。
- 点击后终止 Bundle ID 对应的输入法进程，终止当前用户的 `imklaunchagent`，再通过 `NSWorkspace` 重新打开输入法 App。
- 微信输入法 `WeType` 按同一路径处理，不写死进程名。
- 重启期间禁用按钮；失败时显示错误提醒，成功后重新读取当前输入源。
- 不关闭微信、QQ 等普通应用，也不终止 Finder 扩展。

## 原生软件音量架构

- 删除 SnapClickAudio 虚拟 HAL 驱动和安装/卸载流程。
- macOS 14.2 及以上使用 `AudioHardwareCreateProcessTap` 创建全局立体声 Tap，并排除 SnapClick 自身进程以避免回授。
- Tap 使用 `CATapMutedWhenTapped`，只有 SnapClick 正在读取时才静音原始硬件输出。
- 私有 Aggregate Device 的 IOProc 把 Tap 音频写入无锁环形缓冲区，物理 AUHAL 从缓冲区播放到当前输出设备。
- 软件增益由 C11 原子值保存，在物理输出回调内应用；SnapClick 接管系统音量加、减和静音键。
- 默认输入设备不被读取、修改或重启，麦克风与输入法语音输入保持独立。
- 低于 macOS 14.2 时功能不可启用，设置页显示灰色 12.5px 兼容性说明。

## 音频转发验证

- 物理输出回调在写入有效样本后清除 `OutputIsSilence` 标志。
- Debug 诊断分别统计 Tap 捕获、环形缓冲区读取和物理输出回调的有效峰值及错误状态。
- 自动化测试覆盖 Tap 生命周期、原子软件增益、音量键和输入法重启目标识别。
- 本机运行时验证必须包含：内建麦克风电平、系统听写或第三方语音输入、内建扬声器、BenQ DisplayPort 输出、系统音量键和卸载后的设备状态。

## 失败处理

- Tap 创建失败时保持原始硬件输出不静音并显示错误。
- 找不到当前第三方输入法时隐藏重启按钮，不回退为重启全部输入法。
- 输入法重启失败不阻止音频驱动卸载，但错误必须反馈给用户。
- 在麦克风和直接物理输出通过前，不自动启用软件音量。

## 非目标

- 不提供输入法选择器或输入法安装管理。
- 不重启所有第三方输入法。
- 不修改麦克风权限、听写设置或用户的输入音量。
- 不推送或发布未完成真实硬件验证的音频版本。
