# 系统音量键与音频驱动管理设计

## 目标

SnapClick 通过虚拟音频设备为不支持硬件音量调节的 HDMI/DisplayPort 显示器提供软件音量控制，同时兼容 Mac 内建扬声器。用户使用 macOS 原生音量加、音量减和静音键，不再使用状态栏中的独立音量控件。

## 用户界面

- 删除状态栏菜单中的自定义音量区域。
- 在“设置 → 其他”增加“音频驱动”行。
- 根据 `/Library/Audio/Plug-Ins/HAL/SnapClickAudio.driver` 的实际存在状态显示“已安装”或“未安装”。
- 未安装时显示“安装”按钮；已安装时显示“卸载”按钮。
- 安装或卸载期间禁用按钮并显示处理中状态。
- 操作完成后重新读取磁盘状态并自动切换状态文字和按钮。
- 操作失败时显示错误提醒，并重新读取实际安装状态，避免 UI 与系统状态不一致。

## 音频架构

- SnapClick Audio 继续作为默认虚拟输出设备。
- 虚拟设备的输出流使用 libASPL 的标准输出 `VolumeControl` 和 `MuteControl`。
- macOS 系统音量键直接修改虚拟设备的标准音量/静音属性；libASPL 在输出流进入环形缓冲区前应用增益和静音。
- App 的 HAL AudioUnit 转发器只负责把虚拟设备音频转发到真实输出，不再叠加独立软件增益，避免双重衰减。
- 真实输出可以是 Mac 内建扬声器，也可以是 HDMI/DisplayPort 显示器。切换真实输出时，App 重建转发链并重新把 SnapClick Audio 设为系统默认输出。
- 驱动安装后，SnapClick 启动时自动启用转发链；SnapClick 仍需保持后台运行。

## 安装与卸载

- 安装：验证 App 内嵌驱动签名和 bundle identifier，通过管理员授权复制到系统 HAL 目录，重启 `coreaudiod`，等待虚拟设备出现并启动转发。
- 卸载：先停止转发并把默认输出恢复为真实设备，再通过管理员授权删除系统 HAL 驱动，重启 `coreaudiod`，确认驱动路径消失。
- 用户取消管理员授权按失败处理，不改变推断状态；最终状态始终以驱动路径重新检测为准。

## 兼容性与故障恢复

- 真实输出选择排除 SnapClick 虚拟设备，并复用现有输出设备筛选逻辑。
- 内建扬声器和外接显示器均采用同一 48 kHz、双声道 Float32 转发路径。
- 启动时若虚拟设备仍是默认输出但转发未运行，先恢复真实输出，再重建转发，避免无声状态。
- 卸载或 App 退出前恢复真实输出，避免系统残留到无消费者的虚拟设备。
- 音频设备接入、拔出或默认真实输出变化时，重建转发链并保持虚拟设备为默认输出。

## 验证

- 合约测试验证驱动输出流包含标准音量和静音控制，状态栏不再包含音量菜单项，设置页包含安装状态和安装/卸载操作。
- CoreAudio 查询验证 SnapClick Audio 暴露可写的主输出音量和静音属性。
- 分别使用内建扬声器与外接显示器验证系统音量键、静音键、输出切换、App 重启和驱动卸载。
- 生成签名 Release DMG，并验证 App、Finder 扩展、音频驱动签名、双架构和 DMG 完整性。
